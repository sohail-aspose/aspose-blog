**Australia, Sydney - May 5, 2011 -** [Aspose.Diagram for .NET is a class library for working with Microsoft Visio files][1]. It allows software developers to open VSD and VDX files and manipulate their content from within ASP .NET web applications, web services, and Windows applications.



{{< figure align=center src="images/aspose.diagram-logo2.jpg" alt="Aspose.Diagram icon.">}}


Aspose has a number of products for working with Microsoft Office files on the .NET platform. The most common business application file formats – DOC, DOCX, PPT, PPTX, XLS, XLSX, PDF and so on – can be created and managed using Aspose components. Many of our customers use our products for advanced business applications. Among the file formats they need to be able to manipulate are Visio files. For example, many need to automatically convert Microsoft Visio diagrams to one of any number of popular image formats. Aspose.Diagram for .NET was created to fill this need.

## Efficient and Robust

Aspose.Diagram for .NET is independent of other services and modules and [saves developers from working with Microsoft Automation][2]. It is efficient and robust. The API is extensible, easy to use, and compact. Like all of Aspose’s products, Aspose.Diagram delivers functionality which means that software engineers can write feature-rich applications with as little code as possible. It saves time in development as well as operation.

## Manipulate Diagrams on Server

There are many uses for Aspose.Diagram. It can be used to create server or client-side applications and uses Visio Services features to allow developers to manipulate Visio files on servers. Some of the tasks that developers can perform with Aspose.Diagram are:

*   Convert diagrams to popular image formats for quick and easy information sharing across an organization.
*   Read and change the document properties of a Microsoft Visio diagram for consistent document management.
*   Save diagrams as XML.

The component is quick and easy to install. [Download a trial version of Aspose.Diagram for .NET][3] to see how it can work for you.
