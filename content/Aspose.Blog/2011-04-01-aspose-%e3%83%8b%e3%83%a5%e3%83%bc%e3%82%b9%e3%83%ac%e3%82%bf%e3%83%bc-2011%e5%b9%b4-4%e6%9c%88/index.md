このたびの東北地方太平洋沖地震により、亡くなられた方々のご冥福をお祈り申し上げますとともに、被災された皆様に心よりお見舞い申し上げます。  
皆様方の安全と一日も早い復旧・復興を心よりお祈り申し上げます。

Aspose.Diagram、Aspose.Words Express とカスタマー サービス

![John Owens, the team lead for the Scotland team.][1]Aspose ニュースレターをご購読いただき、誠に有難うございます。

今月は、多くの製品ニュースを用意しております。

まず、Aspose.Diagram for .NET をリリースしました。Aspose.Diagram for .NET は、アプリケーション内から Microsoft Visio ファイルを処理および操作可能なコンポーネントです。Aspose.Total for .NET 製品をお持ちのお客様は、直ぐにこの新製品をダウンロードして使用できます。詳細は、こちらを参照してください。

さらに、[Aspose.Words Express][2] をリリースしました。Aspose.Words Express はフリー ツールで、DOC、DOCX、RTF、HTML おおび ODT ドキュメントを EPUB に変換可能です。変換元のファイルを選択し、変換先を指定して、Convert ボタンを押下するだけです。e-reader で参照したいファイルがあれば、Aspose.Express を使用してください。非常に簡単です。.

[Aspose.Words for .NET を Java に自動移植中です][3]。既に 9700+ ユニットのテストを完了し、各ビルドで問題なく動作しています。まだ移植作業は残っていますが、間もなくリリース予定です。ご期待ください。

カスタマー サービス

対応の良いカスタマー サービスを受けると、それは記憶に残るものです。最近の話ですが、愛用している Macbook Pro が故障したため、Apple Genius Bar へ持って行きました。Apple の担当者は、既知の問題と診断し、既に 3年の保証期間が過ぎてるにも関わらず、マザーボードの取替えなど行っていただき、2日後には、完全に動作する形で Macbook Pro が戻ってきました。Apple の対応に非常に満足しました。

Zappos の CEO Tony Hsieh の _[Delivering Happiness][4]_ を読んだことがあります。彼は、優れたカスタマー サービスを社風とした会社を築きました。結婚式用の靴を花嫁に届けるのに、通常の配送によって万が一届かないリスクを考え、Zappos の従業員がわざわざ手渡しで届けに来たと言う話を聞いたことがあります。

[皆さんが、今まで経験したカスタマー サービスの話をお聞かせください][5]。Aspose 社は、常にお客様の声を聞き、最善の対応を心掛けています。感動的なお話をお待ちしております。

Many thanks,

John Owens, Aspose Scotland Team Leader

製品スポットライト – Aspose.Total が ComponentSource アワードを受賞

[](https://www.aspose.com/)![ComponentSource award badge][7][ComponentsSource では、毎年、ベスト セリング企業と製品を選出しています][8]。Bestselling Publisher Awards と Bestselling Product Awards の二つのカテゴリがあり、Aspose 製品は両方のカテゴリに選出され、4つのアワードを受賞しました。

[Aspose.Total][9] と [Aspose.Words][10] が Top 25 Product アワードを受賞し、[Aspose.Cells][11] が Top 50 Product アワードを受賞しました。  
  
とりわけ Best Selling Publisher Award に選出されたことを光栄に思います。Aspose 製品が広く知れ渡った証拠でもあります。Aspose 製品を支持いただいた皆様にお礼申し上げます。また、2011-2012 もよろしくお願いします。

技術記事 – Web サービスを使用して、Aspose.BarCode for .NET でバーコードを生成

[](https://docs.aspose.com/display/barcodenet/Home)Web サービスで外部のアプリケーションとソフトウェアを統合できます。標準的な request/response モデルを使用して、XML Web サービスを使用できるクライアント アプリケーションには有効です。本技術記事では、Web サービスで Aspose.BarCode for .NET、WinForms クライアントまたは Web クライアントを使用してバーコードを生成する方法を紹介します。

クライアント アプリケーションでは、単純にサーバーに文字列値を送信するだけで (codetext や symbology)、後は Aspose.BarCode が処理します。[記事全文を参照してください][15]。

チュートリアル ビデオ – Aspose.Tasks for .NET で外部タスクの Original ID を検出

[](https://docs.aspose.com/display/tasksnet/Home)アプリケーション内から外部タスクの ID を検出しますか？ Aspose.Tasks for .NET を使用して Microsoft Project で作業する場合には、簡単です。本チュートリアルでは、Microsoft Project の外部タスクを作成し、Visual Studio の ID number を検出するコードの記述を紹介します。本ビデオの再生時間は 2分程です。[ビデオ チュートリアルを参照してください][16]。

移植のヒント - Aspose.Words for .NET 9.2 またはそれ以降へ移植

![](https://www.aspose.com/templates/aspose/App_Themes/V3/images/words/272x272/aspose_words-for-net.png)Aspose.Words for .NET および Java 10 が近日リリース予定です。対象のプロジェクトを最新にして、次のステップに備える良い時期かと思われます。

既に実装済みの API を変更することは非常に面倒な作業となるので、できだけ最小限に変更を抑える予定です。Aspose.Words for .NET 9.2 では、僅かですが API の変更をし、よりオーガナイズされ、Java プラットフォームでの移植作業をより簡素化しました。本記事では、この違いを説明し、コードの更新方法を紹介しています。移植のヒントを参照してください。
