#### Welcome

Welcome to the November 2008 issue of the Aspose Newsletter! In this month’s newsletter, we will provide some introductory information about our spotlight product: Aspose.Total for Reporting Services. We will also look at the new and exciting features offered in the recent releases of Aspose.Cells, Aspose.Slides for JasperReports, Aspose.Flash, Aspose.Form, Aspose.Slides, Aspose.Words, Aspose.BarCode and Aspose.Network. We will also provide information about migrating your code for MS Office Automation to Aspose using Aspose.Network. You will learn about the latest news from Aspose along with the monthly Tech-Tip, which demonstrates how to generate and recognize barcodes for UTF-8 (Unicode) characters, and an introduction to our featured technical article which shows you how to utilize Aspose.Words in other programming languages. In addition to this, we will introduce the latest tutorial video from Aspose, which shows you how to manually configure Aspose.BarCode for Reporting Services with Microsoft SQL Server Reporting Services 2005.

#### Product Spotlight

[](http://www.aspose.com/community/files/50/product-suites/aspose.total.reporting.services/default.aspx)Understanding the special needs of Microsoft SQL Server developers, Aspose introduces [Aspose.Total for Reporting Services][17], a suite of 5 unique Rendering Extensions for Microsoft SQL Server Reporting Services. Using these rendering extensions, developers can easily export their RDL reports to Word, Excel, PowerPoint and PDF documents. Moreover, it is also possible to add Barcodes to the exported documents using Aspose.Total for Reporting Services that can be bought more affordably for one low price! Aspose.Total for Reporting Services is a more reliable and powerful reporting solution that has been produced after extensive research and hard work to enable developers to be more productive by delivering quality solutions to their customers on time. Take the time to [download][18] the free evaluation version today to see how Aspose.Total for Reporting Services can work for you.

#### Enhanced support for XLSX format and improved conversion features in Aspose.Cells for .NET

[](http://www.aspose.com/categories/file-format-components/aspose.cells-for-.net-and-java/default.aspx)[][19]The latest version 4.5.1.0 of [Aspose.Cells][20] for .NET provides improved support for Excel 2007 (XLSX) files. Aspose.Cells for .NET now supports encryption/decryption, read/write charts and shapes, read/write conditional formatting and get/set the cell gradient style in Excel 2007 XLSX files. It also supports reading/writing Data Bar, Color Scale and Icon Set conditional formatting in Excel 2007 XLSX files. The conversion of XLS to PDF, Chart to Image and Worksheet to Image has been significantly improved. In addition to this, enhancements have also been made in printing and copying of Worksheets. Also included in this version are numerous bug fixes and improvements. For more information you can view the official [release page][21] and the relevant blog post. The latest version of Aspose.Cells for .NET can be downloaded from [here][22].

#### Multiple options for setting license in Aspose.Slides for JasperReports

[](http://www.aspose.com/categories/jasperreports-exporters/aspose.slides-for-jasperreports/default.aspx)[Aspose.Slides for JasperReports][23] version 1.2.0.0 has been released. The new release offers three different ways for setting the license. The integration with JasperServer has been improved. It is now possible to define individual template presentation for each report or change default template for all reports. This release also incorporates numerous other improvements and bug fixes. For detailed information you are requested to view the official [release page][24] and accompanying blog post. The latest version of Aspose.Slides for JasperReports can be downloaded from [here][25].

#### Compile ActionScript with flash.\* classes using Aspose.Flash for .NET

[](http://www.aspose.com/categories/file-format-components/aspose.flash-for-.net/default.aspx)With the latest version 1.6.0.0 of [Aspose.Flash][26] for .NET, it is now possible to compile ActionScript with flash.\* classes. This release also supports read/write access for the FlashObject.Id and DefineShape4.ShapeId properties. In addition to this, there are numerous other bug fixes and improvements included in this release. For more information, please consult the official [release page][27] and the relevant blog post. The latest version of Aspose.Flash for .NET can be downloaded from [here][28].

#### Aspose.Form for .NET with enhanced image features

[](http://www.aspose.com/categories/file-format-components/aspose.form-for-.net/default.aspx)[Aspose.Form][29] for .NET version 1.6.0.0 has been released. This new release supports Ink Picture, Inline Image and Linked Image controls in InfoPath 2007 files. Now you can display images with these controls in the InfoPath template. The XmlFormView control has also been improved and now includes some additional functions. For more information, you are requested to view the official [release page][30] and the accompanying blog post. The latest version of Aspose.Form for .NET can be downloaded from [here][31].

#### **New PDF rendering engine in Aspose.Slides for .NET**

[](http://www.aspose.com/categories/file-format-components/aspose.slides-for-.net-and-java/default.aspx)The latest version 2.9.0.0 of [Aspose.Slides][32] for .NET introduces a new PDF rendering engine. The public API was however not changed; you can continue to use the Presentation.SaveToPdf function and utilize the new PDF rendering engine. This release also provides additional cleaning of PPT files (embedded XML) created in PowerPoint 2007. For more details, please consult the official [release page][33] and the relevant blog post. The latest version of Aspose.Slides for .NET can be downloaded from [here][34].

* * *

#### **OpenDocument (ODT) now supported by Aspose.Words for .NET**

[](https://docs.aspose.com/display/wordsjava/Home)[][35][Aspose.Words][36] for .NET version 5.3.0.0 has been released. In this release, the support for OpenDocument (ODT) has been added. It is now possible to load OpenDocument (OpenOffice Writer) documents in Aspose.Words! Aspose.Words will recognize ODT files automatically and also it is easy to convert ODT to OOXML or any other document format supported by Aspose.Words. In addition to this, Smart tags are now supported in DOC, OOXML, RTF and WordML. This release also ships with TLB (COM type library) and IDL (Interface Definition Language) to simplify usage from unmanaged code. Numerous other enhancements and bug fixes have also been included with this release. For more details please consult the official [release page][37] and the accompanying blog post. The latest version of Aspose.Words for .NET can be downloaded from [here][38].

#### **Australia Post barcode supported by Aspose.BarCode for Java**

[](https://docs.aspose.com/display/wordsjava/Home)[][39]The latest version 1.6.0.0 of [Aspose.BarCode][40] for Java now supports the generation and recognition of Australia Post barcode. In this release the barcode recognition process has been optimized. Also included in this release are numerous bug fixes and enhancements. For more information please consult the official [release page][41] and the relevant blog post. The latest version of Aspose.BarCode for Java can be downloaded from [here][42].

* * *

#### **Extensive Office Open XML support in Aspose.Words for Java**

[](https://docs.aspose.com/display/wordsjava/Home)[][43][Aspose.Words][44] for Java version 2.5.0.0 has been released. This release includes a total of 124 fixes and new features. Aspose.Words for Java now supports both macro-enabled and macro-free Microsoft Office 2007 Open XML Document and Template (DOCX, DOCM, DOTX and DOTM). In an effort to have Aspose.Words for Java fully conform to Office Open XML specification we have added support for Smart Tags, OLE embedded and linked objects, VBA projects, and digital signatures for VBA projects. For more details please consult the official [release page][45] and the accompanying blog post. The latest version of Aspose.Words for Java can be downloaded from [here][46].

* * *

#### **Writing Outlook Message Files (\*.msg) with Aspose.Network for .NET**

[](http://www.aspose.com/categories/utility-components/aspose.network-for-.net/default.aspx)The latest version 4.2.0.0 of [Aspose.Network][47] for .NET now supports the creation of Outlook Message Files (\*.msg). Aspose.Network for .NET now allows you to read and write Microsoft Outlook 2003, Microsoft Outlook 2007, and Microsoft Outlook 2008 Message Files (\*.msg). You can also create and delete attachments, add and remove recipients, and convert Email Message Files (\*.eml) to Outlook Message Files (\*.msg). Also included in this release are numerous bug fixes and enhancements. For more information please consult the official [release page][48] and the relevant blog post. The latest version of Aspose.Network for .NET can be downloaded from [here][49].

#### **Technical Tip – How to generate and recognize barcodes for UTF-8 (Unicode) characters**

UTF-8 is a variable length character encoding for Unicode. It can represent any character in Unicode standard. This technical tip shows you how to use [Aspose.BarCode][53] for .NET to generate barcodes for non-English characters e.g. Arabic, Latin, Greek etc. The complete technical tip can be found here.
